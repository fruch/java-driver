void com.datastax.driver.core.SessionStressTest.sessions_should_not_leak_connections ( )

Stress test on opening/closing sessions.

This test opens and closes

Session

in a multithreaded environment and makes sure that there is not connection leak. More specifically, this test performs the following steps:

  • Open 2000
    Session
    concurrently
  • Verify that 2000 sessions are reported as open by the
    Cluster
  • Verify that 4001 connections are reported as open by the
    Cluster
  • Close 1000
    Session
    concurrently
  • Verify that 1000 sessions are reported as open by the
    Cluster
  • Verify that 2001 connections are reported as open by the
    Cluster
  • Open concurrently 1000
    Session
    while 1000 other
    Session
    are closed concurrently
  • Verify that 1000 sessions are reported as open by the
    Cluster
  • Verify that 2001 connections are reported as open by the
    Cluster
  • Close 1000
    Session
    concurrently
  • Verify that 0 sessions are reported as open by the
    Cluster
  • Verify that 1 connection is reported as open by the
    Cluster

This test is linked to JAVA-432.
